# CodeAlpha DevOps Internship - Task 3: Java Application Automation with Gradle

##  Project Overview
This project focuses on **Build Automation** for a Java application using **Gradle**. As part of my DevOps Internship at **CodeAlpha**, I implemented a **Continuous Integration (CI)** pipeline to automate dependency management, compilation, and testing.

## Tech Stack
*   **Language:** Java (JDK 17)
*   **Build Tool:** Gradle (Groovy DSL)
*   **CI/CD:** GitHub Actions
*   **Version Control:** GitHub

## DevOps Workflow (Automation)
The project uses **GitHub Actions** to trigger an automated build process on every push to the repository. This ensures that the code is always "buildable" and free from compilation errors.

### Key Automation Steps:
1.  **Environment Setup:** Automatically configures the Java environment (JDK 17).
2.  **Dependency Resolution:** Gradle fetches all necessary libraries defined in `build.gradle`.
3.  **Compilation:** The CI pipeline runs `./gradlew build` to compile the source code.
4.  **Verification:** Ensures that the build is successful before merging any changes.

##  Project Structure
*   `app/src/main/java/`: Source code of the Java application.
*   `build.gradle`: Configuration for dependencies and build tasks.
*   `.github/workflows/gradle.yml`: Definition of the CI pipeline.

You can verify the successful execution of this task by checking the **Actions** tab in this repository. A green checkmark (✅) indicates that the **"BUILD SUCCESSFUL"** log was generated by Gradle in the cloud environment.
